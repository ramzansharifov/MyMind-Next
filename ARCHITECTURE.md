# Архитектура MyMind

MyMind — локальное Electron-приложение. Renderer построен на React, main process владеет SQLite и файловым хранилищем, а preload публикует узкий типизированный IPC API.

## Границы процессов

- `src/main` — создание окна, миграции, репозитории, файловые операции и политики безопасности.
- `src/preload` — единственный мост renderer → main. `contextIsolation`, `sandbox` включены, Node.js в renderer выключен.
- `src/renderer` — интерфейс и локальное состояние редактирования. Модули описаны в `app/module-registry.ts` и загружаются лениво.
- `src/shared` — контракты IPC, Zod-схемы и безопасные платформонезависимые утилиты.

## Данные и сохранение

Материалы хранятся в SQLite как версионированный документ. Renderer последовательно ставит autosave в очередь; main дополнительно сериализует сохранение, удаление и очистку ресурсов по material ID. После транзакции очистка получает именно сохранённый документ, поэтому более старый запрос не может удалить ресурс новой версии.

SQLite работает в WAL, с foreign keys и `busy_timeout`. Миграции Drizzle запускаются до регистрации рабочего окна. Изменение схемы требует `npm run db:generate`, проверки с существующей копией базы и `npm run db:check`.

## Локальные ресурсы

Файлы лежат под каталогом user data и выдаются через `mymind-asset://`. URL канонизируется, имена и ID валидируются, путь обязан остаться внутри каталога конкретного материала. Сохранение проверяет существование файла и совпадение владельца. Протокол поддерживает только GET/HEAD и byte ranges.

## Расширение

Новый верхнеуровневый модуль добавляется одной записью в `app/module-registry.ts`; навигация строится из того же реестра. Новые типы study-блоков должны одновременно получить shared-контракт и Zod-схему, фабрику документа, read/edit renderer, settings UI, извлечение plain text/ссылочных целей и тесты совместимости.
