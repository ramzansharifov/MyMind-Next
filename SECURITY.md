# Безопасность

## Сообщение об уязвимости

Не публикуйте эксплойт в открытом issue. Передайте владельцу репозитория описание, версию приложения, шаги воспроизведения и оценку влияния приватным каналом GitHub. Не прикладывайте реальные пользовательские базы или личные файлы.

## Модель угроз

Приложение считает содержимое материалов, импортированные файлы, Markdown/Mermaid и внешние URL недоверенными. Renderer не получает Node.js API. Все входы IPC валидируются Zod в main process; доступ к файлам остаётся в main. Новые окна запрещены, а допустимые HTTPS/mailto ссылки передаются системному браузеру.

Production CSP не содержит localhost, websocket и `unsafe-eval`. Dev-исключения добавляются только response headers локального dev server. Запросы camera, microphone, geolocation, notifications, MIDI, pointer lock, clipboard-read и fullscreen отклоняются централизованной permission policy.

## Хранение и резервные копии

Данные находятся в Electron `userData`: `data/mymind.sqlite` и каталог study assets. Для согласованной ручной копии полностью закройте приложение и скопируйте весь `userData`, включая возможные `-wal`/`-shm`, а не один файл SQLite. Восстанавливайте только в совместимую или более новую версию приложения и сначала сохраните отдельную копию текущих данных.

Удаление узла необратимо на уровне UI. Ошибка очистки файлов логируется и не откатывает уже подтверждённую транзакцию БД; такой остаточный файл безопаснее потери данных и может быть очищен позднее.

## Релизы

Сборка не равна доверенному релизу. Публичные пакеты должны быть подписаны сертификатом платформы, а macOS-сборки — notarized. Секреты подписи хранятся только в защищённом CI. Автообновление нельзя включать без подписанного канала, HTTPS и проверки издателя. Неподписанные локальные артефакты предназначены только для разработки.
